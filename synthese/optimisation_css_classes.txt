OPTIMISATION CSS - RENOMMAGE DES CLASSES POUR CLARIFICATION PÉDAGOGIQUE
==========================================================================

OBJECTIF :
Éliminer la confusion causée par les classes CSS identiques dans différents fichiers.
Chaque composant React aura ses propres noms de classes CSS pour une meilleure compréhension.

PROBLÈME IDENTIFIÉ :
- Plusieurs fichiers CSS utilisaient les mêmes noms de classes (.cart-item, .page-header)
- Confusion  : quel fichier CSS utilise quel composant ?
- Conflits potentiels de styles

SOLUTION APPLIQUÉE :
Renommer les classes pour qu'elles soient uniques à chaque composant.

CHANGEMENTS EFFECTUÉS :

1. CheckoutPage.css
   - .page-header → .checkout-header
   - .page-header h2 → .checkout-header h2
   - Mis à jour dans les media queries également

2. HomePage.css  
   - .page-header → .home-header
   - .page-header h2 → .home-header h2
   - .page-header p → .home-header p

3. CartPage.css
   - Déjà bien nommé avec .cart-page-header (aucun changement nécessaire)

4. Cart.css
   - Garde .cart-item (c'est le composant principal pour les items du panier)
   - .cart-header reste inchangé

5. ShippingPage.css
   - Aucun changement nécessaire (déjà bien nommé avec .shipping-form)

6. SUPPRESSION DE FICHIER INUTILE
   - CartPage_new.css supprimé (non utilisé dans le projet)

MISE À JOUR REQUISE DANS LES COMPOSANTS REACT :
- CheckoutPage.jsx : className="page-header" → className="checkout-header" ✅ FAIT
- HomePage.jsx : Aucun changement nécessaire (n'utilise pas les classes renommées) ✅ VÉRIFIÉ
- CartPage.jsx : Aucun changement (déjà correct avec .cart-page-header) ✅ VÉRIFIÉ

AVANTAGES PÉDAGOGIQUES :
- Chaque fichier CSS correspond clairement à un composant
- Plus de confusion sur "où se trouve ce style"
- Facilite le debugging pour les étudiants
- Enseigne les bonnes pratiques de nommage CSS
- Prépare à l'utilisation future des CSS Modules

CORRESPONDANCE COMPOSANT ↔ FICHIER CSS :
- Cart.jsx → Cart.css (.cart-*)
