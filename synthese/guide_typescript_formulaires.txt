GUIDE TYPESCRIPT : FORMULAIRES REACT TYPÉS
==========================================

PRÉREQUIS : Avoir lu et pratiqué le guide_rapide_formulaires_react_v2.txt

OBJECTIF : Découvrir comment TypeScript améliore la robustesse et la 
maintenance des formulaires React en détectant les erreurs avant l'exécution.

===============================================================================

POURQUOI TYPESCRIPT AVEC LES FORMULAIRES ?
==========================================

VOUS MAÎTRISEZ DÉJÀ : Les formulaires React en JavaScript pur. Maintenant 
nous allons voir comment TypeScript transforme cette expérience en apportant 
sécurité et autocomplétion.

PROBLÈMES COURANTS EN JAVASCRIPT PUR :
- Erreurs de frappe dans les noms de propriétés (fistName au lieu de firstName)
- Oubli de propriétés obligatoires dans les props
- Valeurs inattendues dans les fonctions de validation
- Bugs découverts seulement à l'exécution

AVANTAGES DE TYPESCRIPT :
- Détection d'erreurs pendant l'écriture du code
- Autocomplétion intelligente dans l'éditeur
- Documentation automatique des structures de données
- Refactoring sécurisé (renommage, modification de structure)

TRANSITION DOUCE : TypeScript, c'est du JavaScript avec des annotations de types. 
Votre logique de formulaires reste exactement la même !

===============================================================================

1. TYPAGE DES DONNÉES DE FORMULAIRE
===================================

CE QU'ON VA APPRENDRE : Comment définir précisément la structure de nos données 
de formulaire pour éliminer les erreurs de propriétés manquantes ou mal nommées.

Définition du type FormData
---------------------------

AVANT EN JAVASCRIPT :
const [formData, setFormData] = useState({
    firstName: '',
    lastName: '',
    email: '',
    // Risque d'oubli, erreurs de frappe...
});

APRÈS EN TYPESCRIPT :
interface ShippingFormData {
    firstName: string;
    lastName: string;
    email: string;
    phone?: string;
    address: string;
    city: string;
    postalCode: string;
    country: string;
    deliveryOptions: 'standard' | 'express';
    newsletter: boolean;
}

const [formData, setFormData] = useState<ShippingFormData>({
    firstName: '',
    lastName: '',
    email: '',
    phone: '',
    address: '',
    city: '',
    postalCode: '',
    country: 'France',
    deliveryOptions: 'standard',
    newsletter: false
});

BÉNÉFICES IMMÉDIATS :
- AUTOCOMPLÉTION : Votre éditeur propose automatiquement firstName, lastName, etc.
- VALIDATION : Impossible d'écrire formData.fistName par erreur
- UNION TYPES : deliveryOptions ne peut être que 'standard' ou 'express'
- PROPRIÉTÉS OPTIONNELLES : phone? indique que ce champ peut être vide

TYPES UNION : 'standard' | 'express' limite les valeurs possibles. Plus de 
risque d'assigner une valeur invalide comme 'rapide' ou 'lent'.

2. TYPAGE DES HANDLERS ET ÉVÉNEMENTS
====================================

CE QU'ON VA APPRENDRE : Comment typer correctement les fonctions qui gèrent 
les événements pour bénéficier de l'autocomplétion et éviter les erreurs.

Handler avec types explicites
-----------------------------

AVANT EN JAVASCRIPT :
const handleChange = (e) => {
    const { name, value, type, checked } = e.target;
    // e.target pourrait être n'importe quoi...
};

APRÈS EN TYPESCRIPT :
const handleChange = (e: React.ChangeEvent<HTMLInputElement | HTMLSelectElement>) => {
    const { name, value, type, checked } = e.target;
    setFormData(prev => ({
        ...prev,
        [name]: type === 'checkbox' ? checked : value
    }));
};

TYPES D'ÉVÉNEMENTS REACT :
- React.ChangeEvent<HTMLInputElement> pour les inputs
- React.ChangeEvent<HTMLSelectElement> pour les selects
- React.ChangeEvent<HTMLTextAreaElement> pour les textareas
- React.FormEvent<HTMLFormElement> pour la soumission

Handler pour soumission typé
----------------------------

const handleSubmit = (e: React.FormEvent<HTMLFormElement>) => {
    e.preventDefault();
    
    setIsSubmitting(true);
    setErrors({});
    
    try {
        const validationErrors = validateForm(formData);
        // ... reste identique
    } catch (error) {
        setErrors({ general: 'Erreur lors de l\'envoi' });
    }
};

AVANTAGE : TypeScript vous garantit que e.preventDefault() existe et que 
vous appelez la bonne méthode sur le bon type d'événement.

===============================================================================

3. VALIDATION TYPÉE
===================

CE QU'ON VA APPRENDRE : Comment créer des fonctions de validation robustes 
qui garantissent la cohérence entre les données attendues et les données reçues.

Interface pour les erreurs
--------------------------

type ValidationErrors = Partial<Record<keyof ShippingFormData, string>> & {
    general?: string;
};

const [errors, setErrors] = useState<ValidationErrors>({});

EXPLICATION DU TYPE :
- Partial<Record<keyof ShippingFormData, string>> : chaque propriété de 
  ShippingFormData peut avoir une erreur (string) mais c'est optionnel
- & { general?: string } : on ajoute la possibilité d'une erreur générale
- keyof récupère automatiquement les noms des propriétés

Fonction de validation typée
----------------------------

const validateForm = (data: ShippingFormData): ValidationErrors => {
    const errors: ValidationErrors = {};

    if (!data.firstName.trim()) errors.firstName = 'Prénom requis';
    if (!data.lastName.trim()) errors.lastName = 'Nom requis';
    if (!data.email.trim()) errors.email = 'Email requis';
    
    const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
    if (data.email && !emailRegex.test(data.email)) {
        errors.email = 'Format email invalide';
    }
    
    const postalRegex = /^[0-9]{5}$/;
    if (data.postalCode && !postalRegex.test(data.postalCode)) {
        errors.postalCode = 'Code postal invalide';
    }

    return errors;
};

BÉNÉFICES DU TYPAGE :
- PARAMÈTRE GARANTI : data est forcément de type ShippingFormData
- RETOUR COHÉRENT : la fonction retourne forcément ValidationErrors
- AUTOCOMPLÉTION : data.firstName, data.email sont proposés automatiquement
- SÉCURITÉ : Impossible d'accéder à data.inexistant par erreur

4. TYPAGE DES PROPS DU COMPOSANT
================================

CE QU'ON VA APPRENDRE : Comment définir précisément les propriétés que votre 
composant attend pour éliminer les erreurs d'utilisation.

Interface des props
-------------------

interface ShippingFormProps {
    onShippingComplete?: (data: ShippingFormData) => void;
    initialData?: Partial<ShippingFormData>;
    isReadonly?: boolean;
    theme?: 'light' | 'dark';
}

function ShippingForm({ 
    onShippingComplete, 
    initialData,
    isReadonly = false,
    theme = 'light'
}: ShippingFormProps) {
    // ...
}

BÉNÉFICES DES PROPS TYPÉES :
- UTILISATION SÉCURISÉE : <ShippingForm onShippingComplete={handleData} />
- VALEURS PAR DÉFAUT : isReadonly = false directement dans la destructuration
- CALLBACK TYPÉ : onShippingComplete reçoit forcément ShippingFormData
- DOCUMENTATION VIVANTE : les types documentent l'utilisation du composant

Utilisation avec initialData
-----------------------------

const [formData, setFormData] = useState<ShippingFormData>({
    firstName: initialData?.firstName || '',
    lastName: initialData?.lastName || '',
    email: initialData?.email || '',
    phone: initialData?.phone || '',
    address: initialData?.address || '',
    city: initialData?.city || '',
    postalCode: initialData?.postalCode || '',
    country: initialData?.country || 'France',
    deliveryOptions: initialData?.deliveryOptions || 'standard',
    newsletter: initialData?.newsletter || false
});

PARTIAL<T> : Ce type utilitaire rend toutes les propriétés optionnelles. 
Parfait pour initialData où on ne veut pas forcer tous les champs.

OPTIONAL CHAINING : initialData?.firstName évite les erreurs si initialData 
est undefined.

5. COMPOSANT COMPLET AVEC TYPESCRIPT
====================================

CE QU'ON VA VOIR : Le même composant ShippingForm que dans le guide JavaScript, 
mais avec tous les types TypeScript pour une robustesse maximale.

import React, { useState } from 'react';

interface ShippingFormData {
    firstName: string;
    lastName: string;
    email: string;
    phone?: string;
    address: string;
    city: string;
    postalCode: string;
    country: string;
    deliveryOptions: 'standard' | 'express';
    newsletter: boolean;
}

type ValidationErrors = Partial<Record<keyof ShippingFormData, string>> & {
    general?: string;
};

interface ShippingFormProps {
    onShippingComplete?: (data: ShippingFormData) => void;
    initialData?: Partial<ShippingFormData>;
    isReadonly?: boolean;
}

const ShippingForm: React.FC<ShippingFormProps> = ({ 
    onShippingComplete, 
    initialData,
    isReadonly = false
}) => {
    const [formData, setFormData] = useState<ShippingFormData>({
        firstName: initialData?.firstName || '',
        lastName: initialData?.lastName || '',
        email: initialData?.email || '',
        phone: initialData?.phone || '',
        address: initialData?.address || '',
        city: initialData?.city || '',
        postalCode: initialData?.postalCode || '',
        country: initialData?.country || 'France',
        deliveryOptions: initialData?.deliveryOptions || 'standard',
        newsletter: initialData?.newsletter || false
    });

    const [isSubmitting, setIsSubmitting] = useState<boolean>(false);
    const [errors, setErrors] = useState<ValidationErrors>({});
    const [success, setSuccess] = useState<boolean>(false);

    const handleChange = (e: React.ChangeEvent<HTMLInputElement | HTMLSelectElement>) => {
        const { name, value, type } = e.target;
        const checked = (e.target as HTMLInputElement).checked;
        
        setFormData(prev => ({
            ...prev,
            [name]: type === 'checkbox' ? checked : value
        }));
        
        if (errors[name as keyof ValidationErrors]) {
            setErrors(prev => ({ ...prev, [name]: undefined }));
        }
    };

    const validateForm = (data: ShippingFormData): ValidationErrors => {
        const errors: ValidationErrors = {};

        if (!data.firstName.trim()) errors.firstName = 'Prénom requis';
        if (!data.lastName.trim()) errors.lastName = 'Nom requis';
        if (!data.email.trim()) errors.email = 'Email requis';
        if (!data.address.trim()) errors.address = 'Adresse requise';
        if (!data.city.trim()) errors.city = 'Ville requise';
        if (!data.postalCode.trim()) errors.postalCode = 'Code postal requis';

        const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
        if (data.email && !emailRegex.test(data.email)) {
            errors.email = 'Format email invalide';
        }

        const postalRegex = /^[0-9]{5}$/;
        if (data.postalCode && !postalRegex.test(data.postalCode)) {
            errors.postalCode = 'Code postal invalide';
        }

        return errors;
    };

    const handleSubmit = async (e: React.FormEvent<HTMLFormElement>) => {
        e.preventDefault();
        
        setIsSubmitting(true);
        setErrors({});
        
        try {
            const validationErrors = validateForm(formData);
            if (Object.keys(validationErrors).length > 0) {
                setErrors(validationErrors);
                return;
            }
            
            await new Promise(resolve => setTimeout(resolve, 1000));
            
            setSuccess(true);
            onShippingComplete?.(formData);
            
        } catch (error) {
            setErrors({ general: 'Erreur lors de l\'envoi' });
        } finally {
            setIsSubmitting(false);
        }
    };

    if (success) {
        return (
            <div className="success-message">
                <h2>Commande confirmée !</h2>
                <p>Vos informations de livraison ont été enregistrées.</p>
            </div>
        );
    }

    return (
        <div className="shipping-form">
            <h2>Informations de livraison</h2>
            
            <form onSubmit={handleSubmit}>
                <fieldset>
                    <legend>Informations personnelles</legend>
                    
                    <div className="form-group">
                        <label htmlFor="firstName">Prénom *</label>
                        <input 
                            id="firstName"
                            name="firstName"
                            type="text"
                            value={formData.firstName}
                            onChange={handleChange}
                            disabled={isSubmitting || isReadonly}
                        />
                        {errors.firstName && 
                            <span className="error-text">{errors.firstName}</span>
                        }
                    </div>
                    
                    <div className="form-group">
                        <label htmlFor="email">Email *</label>
                        <input 
                            id="email"
                            name="email"
                            type="email"
                            value={formData.email}
                            onChange={handleChange}
                            disabled={isSubmitting || isReadonly}
                        />
                        {errors.email && 
                            <span className="error-text">{errors.email}</span>
                        }
                    </div>
                </fieldset>

                <fieldset>
                    <legend>Options de livraison</legend>
                    
                    <div className="radio-group">
                        <label>
                            <input 
                                name="deliveryOptions"
                                type="radio"
                                value="standard"
                                checked={formData.deliveryOptions === 'standard'}
                                onChange={handleChange}
                                disabled={isSubmitting || isReadonly}
                            />
                            Livraison standard (3-5 jours) - Gratuit
                        </label>
                        
                        <label>
                            <input 
                                name="deliveryOptions"
                                type="radio"
                                value="express"
                                checked={formData.deliveryOptions === 'express'}
                                onChange={handleChange}
                                disabled={isSubmitting || isReadonly}
                            />
                            Livraison express (24h) - 9,99€
                        </label>
                    </div>
                </fieldset>

                {errors.general && (
                    <div className="error-banner">
                        {errors.general}
                    </div>
                )}

                <button 
                    type="submit" 
                    disabled={isSubmitting || isReadonly}
                >
                    {isSubmitting ? 'Traitement...' : 'Confirmer la commande'}
                </button>
            </form>
        </div>
    );
};

export default ShippingForm;

POINTS CLÉS TYPESCRIPT :
- React.FC<Props> : Type pour les composants fonctionnels React
- Type assertions : (e.target as HTMLInputElement).checked pour les checkboxes
- Optionnel chaining : onShippingComplete?.() pour les callbacks optionnels
- keyof operator : name as keyof ValidationErrors pour l'accès sécurisé

6. TYPES UTILITAIRES AVANCÉS
============================

CE QU'ON VA APPRENDRE : Des types TypeScript avancés qui rendent vos 
formulaires encore plus robustes et réutilisables.

Type pour les champs requis seulement
-------------------------------------

type RequiredFields = Pick<ShippingFormData, 'firstName' | 'lastName' | 'email' | 'address'>;

function validateRequiredFields(data: Partial<ShippingFormData>): RequiredFields | null {
    const requiredData = {
        firstName: data.firstName || '',
        lastName: data.lastName || '',
        email: data.email || '',
        address: data.address || ''
    };
    
    const hasAllRequired = Object.values(requiredData).every(value => value.trim());
    return hasAllRequired ? requiredData : null;
}

Type pour les étapes de formulaire
----------------------------------

type FormStep = 'personal' | 'address' | 'delivery' | 'confirmation';

interface FormStepProps {
    currentStep: FormStep;
    onStepChange: (step: FormStep) => void;
    formData: ShippingFormData;
}

Type générique pour la validation
---------------------------------

type Validator<T> = (data: T) => Partial<Record<keyof T, string>>;

const createValidator = <T,>(rules: Validator<T>): Validator<T> => {
    return (data: T) => rules(data);
};

const shippingValidator = createValidator<ShippingFormData>((data) => {
    const errors: Partial<Record<keyof ShippingFormData, string>> = {};
    
    if (!data.firstName.trim()) errors.firstName = 'Prénom requis';
    if (!data.email.includes('@')) errors.email = 'Email invalide';
    
    return errors;
});

AVANTAGES DES TYPES AVANCÉS :
- RÉUTILISABILITÉ : createValidator peut servir pour tout type de formulaire
- COMPOSITION : Pick, Partial permettent de créer des variantes de types
- ÉVOLUTIVITÉ : Modifier ShippingFormData met automatiquement à jour tous les types dérivés

7. CONFIGURATION TYPESCRIPT POUR REACT
======================================

CE QU'ON VA APPRENDRE : Comment configurer correctement TypeScript dans un 
projet React pour avoir la meilleure expérience de développement.

Installation des dépendances
----------------------------

npm install typescript @types/react @types/react-dom

tsconfig.json recommandé
------------------------

{
  "compilerOptions": {
    "target": "es5",
    "lib": ["dom", "dom.iterable", "esnext"],
    "allowJs": true,
    "skipLibCheck": true,
    "esModuleInterop": true,
    "allowSyntheticDefaultImports": true,
    "strict": true,
    "forceConsistentCasingInFileNames": true,
    "moduleResolution": "node",
    "resolveJsonModule": true,
    "isolatedModules": true,
    "noEmit": true,
    "jsx": "react-jsx"
  },
  "include": ["src"]
}

Extensions de fichiers
----------------------

- .tsx pour les composants React avec JSX
- .ts pour les utilitaires et hooks sans JSX
- Renommer progressivement vos fichiers .js en .ts ou .tsx

Transition progressive
---------------------

1. COMMENCER : Renommer un seul composant en .tsx
2. AJOUTER : Les types petit à petit (props, state, handlers)
3. ÉTENDRE : Types plus complexes quand vous êtes à l'aise
4. GÉNÉRALISER : Types réutilisables pour plusieurs composants

COMPARAISON AVANT/APRÈS
=======================

ERREURS DÉTECTÉES AUTOMATIQUEMENT :

JavaScript (erreur à l'exécution) :
formData.fistName = 'Jean';  // Erreur de frappe
onComplete();               // Oubli du paramètre
setFormData('invalid');     // Mauvais type

TypeScript (erreur pendant le développement) :
formData.fistName = 'Jean';    // ERREUR : Property 'fistName' does not exist
onComplete();                  // ERREUR : Expected 1 arguments, but got 0
setFormData('invalid');        // ERREUR : Argument of type 'string' is not assignable

AUTOCOMPLÉTION AMÉLIORÉE :
- Tapez formData. et voyez toutes les propriétés disponibles
- Suggestions automatiques pour les valeurs des union types
- Documentation des paramètres directement dans l'éditeur

POINTS CLÉS À RETENIR
=====================

CONCEPTS MAÎTRISÉS :
- INTERFACES : Définir la structure des données de formulaire
- TYPES UNION : Limiter les valeurs possibles ('standard' | 'express')
- GÉNÉRIQUES : Créer des types réutilisables <T>
- TYPE ASSERTIONS : Gérer les cas où TypeScript ne peut pas inférer
- PROPS TYPÉES : Sécuriser l'utilisation des composants

QUAND UTILISER TYPESCRIPT :
- Projets avec plusieurs développeurs
- Applications complexes avec beaucoup de formulaires
- Besoin de refactoring fréquent
- Volonté de détecter les erreurs tôt

PROGRESSION RECOMMANDÉE :
1. Maîtriser d'abord React en JavaScript
2. Ajouter TypeScript sur un projet simple
3. Étendre progressivement le typage
4. Utiliser les types avancés pour les gros projets

TypeScript transforme votre expérience de développement React en apportant 
confiance et productivité. Une fois adopté, difficile de revenir en arrière !
