1. INTRODUCTION - POURQUOI LES FORMULAIRES SONT CRUCIAUX
========================================================

OBJECTIFS D'APPRENTISSAGE :
- Comprendre le rôle central des formulaires dans les applications web
- Identifier les spécificités React vs HTML vanilla
- Maîtriser le concept de "controlled components"

Pour débuter ce parcours d'apprentissage, il est essentiel de comprendre pourquoi
les formulaires occupent une place si importante dans le développement React moderne.

Les formulaires dans le développement moderne :

Dans toute application web interactive, les formulaires constituent le pont principal 
entre l'utilisateur et votre système. Ils permettent :
- COLLECTE DE DONNÉES : Inscription, connexion, commandes e-commerce
- INTERACTIVITÉ : Barres de recherche, filtres dynamiques, paramétrages
- VALIDATION : Contrôle en temps réel de la qualité des données saisies  
- EXPÉRIENCE UTILISATEUR : Feedback immédiat et guidage intuitif

Cette centralité explique pourquoi React a développé une approche spécifique 
pour gérer les formulaires, différente du HTML traditionnel.

REACT VS HTML TRADITIONNEL - Une évolution nécessaire :

Alors que le HTML classique laisse le navigateur gérer l'état des champs,
React introduit une philosophie de contrôle total qui transforme notre approche.

HTML traditionnel (Uncontrolled) :
<form>
    <input type="text" name="username" />
    <input type="email" name="email" />
    <button type="submit">Envoyer</button>
</form>

Dans cette approche traditionnelle, le navigateur stocke les valeurs et nous
n'y avons acces qu'au moment de la soumission du formulaire.

React moderne (Controlled Component) :
function MyForm() {
    const [username, setUsername] = useState('');
    const [email, setEmail] = useState('');
    
    return (
        <form onSubmit={handleSubmit}>
            <input 
                type="text" 
                value={username}
                onChange={(e) => setUsername(e.target.value)}
            />
            <input 
                type="email" 
                value={email}
                onChange={(e) => setEmail(e.target.value)}
            />
            <button type="submit">Envoyer</button>
        </form>
    );
}

Cette évolution vers les "controlled components" apporte des avantages décisifs :
- React contrôle totalement l'état à tout moment
- Validation en temps réel pendant la saisie
- Logique centralisée et comportement prévisible
- Tests et debugging considérablement facilités

Maintenant que nous comprenons cette philosophie générale, explorons les principes
fondamentaux qui rendent cette approche si puissante.

2. PRINCIPES FONDAMENTAUX - Les mécanismes sous-jacents
==========================

Après avoir saisi l'importance des controlled components, il est temps d'analyser 
les mécanismes techniques qui les rendent possibles. Cette section détaille les
concepts essentiels que vous devez absolument maîtriser.

A) CONTROLLED VS UNCONTROLLED COMPONENTS - Le choix architectural décisif

Cette distinction fondamentale détermine toute l'architecture de vos formulaires.
Comprendre cette différence vous permettra de faire les bons choix techniques.

APPROCHE UNCONTROLLED - Quand React ne contrôle pas :
function UncontrolledForm() {
    const inputRef = useRef();
    
    const handleSubmit = () => {
        console.log(inputRef.current.value); // Accès via DOM uniquement
    };
    
    return <input ref={inputRef} type="text" />;
}

Dans cette approche, React n'a aucune connaissance de la valeur courante.
Nous devons interroger le DOM pour récupérer les données, ce qui limite 
grandement nos possibilités d'interaction.

APPROCHE CONTROLLED - La puissance du contrôle total :
function ControlledForm() {
    const [value, setValue] = useState('');
    
    return (
        <input 
            type="text" 
            value={value}
            onChange={(e) => setValue(e.target.value)}
        />
    );
}

Ici, React connaît à tout moment la valeur exacte du champ. Cette connaissance
ouvre la porte à la validation temps réel, aux transformations automatiques,
et à une logique applicative sophistiquée.

B) LE PATTERN VALUE + ONCHANGE - Le socle technique incontournable

Cette combinaison constitue le fondement de tous vos formulaires React.
Une fois ce pattern maîtrisé, vous pourrez l'appliquer à n'importe quel type
de champ de formulaire.

RÈGLE FONDAMENTALE : Chaque input contrôlé requiert impérativement :
- value : La valeur actuelle provenant de l'état React
- onChange : La fonction qui met à jour cet état

Implémentation du pattern de base :
const [inputValue, setInputValue] = useState('');

<input 
    value={inputValue}                                   // État actuel
    onChange={(e) => setInputValue(e.target.value)}     // Mise à jour
/>

Cette liaison bidirectionnelle garantit que React et le DOM restent
parfaitement synchronises a tout moment.

C) L'OBJET EVENT ET TARGET - Exploiter toute la richesse des événements

Pour tirer pleinement parti des controlled components, vous devez comprendre
les informations disponibles dans l'objet événement. Ces données vous
permettront de créer des interactions sophistiquées.

const handleChange = (event) => {
    console.log(event.target.value);    // Nouvelle valeur saisie
    console.log(event.target.name);     // Attribut name du champ
    console.log(event.target.type);     // Type d'input (text, email, etc.)
    console.log(event.target.checked);  // État pour les checkboxes/radio
};

Cette richesse d'informations devient particulièrement précieuse quand nous
gerons plusieurs champs simultanement, comme nous le verrons dans la section
suivante sur la syntaxe pratique.

Avec ces fondements solidement etablis, nous pouvons maintenant aborder 
l'implementation concrete de differents types d'elements de formulaire.

3. SYNTAXE DE BASE - De la theorie a la pratique
=============================================

Maintenant que vous maîtrisez les principes fondamentaux, il est temps de les
appliquer concretement. Cette section vous guide pas a pas dans l'implementation
de tous les types d'elements de formulaire que vous rencontrerez.

A) INPUT TEXTE SIMPLE - Premier pas vers la maîtrise

Commençons par l'element le plus courant : l'input texte. Cet exemple illustre
parfaitement le pattern value/onChange que nous venons d'etudier.

function BasicTextInput() {
    const [text, setText] = useState('');
    
    return (
        <div>
            <label htmlFor="basic-input">Votre nom :</label>
            <input 
                id="basic-input"
                type="text"
                value={text}
                onChange={(e) => setText(e.target.value)}
                placeholder="Entrez votre nom"
            />
            <p>Vous avez tape : {text}</p>
        </div>
    );
}

Notez l'ajout du feedback immediat avec "Vous avez tape : {text}". Cette
capacité à afficher la valeur en temps réel illustre parfaitement la puissance
des controlled components.

B) DIFFERENTS TYPES D'INPUTS - Elargir votre palette d'outils

Une fois le principe de base assimile, vous pouvez l'etendre a tous les types
d'inputs HTML. Chaque type a ses spécificités, mais le pattern reste identique.

function AllInputTypes() {
    const [formData, setFormData] = useState({
        text: '',
        email: '',
        password: '',
        number: '',
        date: '',
        checkbox: false,
        radio: '',
        select: '',
        textarea: ''
    });
    
    const handleChange = (e) => {
        const { name, value, type, checked } = e.target;
        setFormData(prev => ({
            ...prev,
            [name]: type === 'checkbox' ? checked : value
        }));
    };
    
    return (
        <form>
            // Texte standard
            <input 
                name="text" 
                type="text" 
                value={formData.text}
                onChange={handleChange}
            />
            
            // Email avec validation navigateur
            <input 
                name="email" 
                type="email" 
                value={formData.email}
                onChange={handleChange}
            />
            
            // Mot de passe masque
            <input 
                name="password" 
                type="password" 
                value={formData.password}
                onChange={handleChange}
            />
            
            // Nombre avec contrôles intégrés
            <input 
                name="number" 
                type="number" 
                value={formData.number}
                onChange={handleChange}
            />
            
            // Date avec selecteur visuel
            <input 
                name="date" 
                type="date" 
                value={formData.date}
                onChange={handleChange}
            />
            
            // Checkbox - Attention : utilise checked au lieu de value !
            <input 
                name="checkbox" 
                type="checkbox" 
                checked={formData.checkbox}
                onChange={handleChange}
            />
            
            // Radio buttons - Gestion d'options exclusives
            <input 
                name="radio" 
                type="radio" 
                value="option1"
                checked={formData.radio === 'option1'}
                onChange={handleChange}
            />
            
            // Liste deroulante
            <select 
                name="select" 
                value={formData.select}
                onChange={handleChange}
            >
                <option value="">Choisir...</option>
                <option value="option1">Option 1</option>
                <option value="option2">Option 2</option>
            </select>
            
            // Zone de texte multiligne
            <textarea 
                name="textarea" 
                value={formData.textarea}
                onChange={handleChange}
            />
        </form>
    );
}

Cette diversite d'inputs partage le meme principe, mais observez les subtilites :
- Les checkboxes utilisent "checked" au lieu de "value"
- Les radio buttons nécessitent une comparaison pour déterminer leur état
- Tous beneficient du meme handler handleChange unifie

C) GESTION CENTRALISÉE AVEC UN OBJET - L'approche professionnelle

Plutôt que de créer un useState pour chaque champ (ce qui devient vite ingérable),
la technique recommandée consiste à grouper toutes les données dans un objet unique.
Cette approche facilite la maintenance et prepare l'integration avec des APIs.

TECHNIQUE RECOMMANDÉE - Un seul état pour tout le formulaire :
function CentralizedForm() {
    const [formData, setFormData] = useState({
        firstName: '',
        lastName: '',
        email: '',
        phone: ''
    });
    
    // Handler generique pour tous les inputs
    const handleChange = (e) => {
        const { name, value } = e.target;
        setFormData(prevData => ({
            ...prevData,
            [name]: value
        }));
    };
    
    return (
        <form>
            <input 
                name="firstName"
                value={formData.firstName}
                onChange={handleChange}
                placeholder="Prenom"
            />
            <input 
                name="lastName"
                value={formData.lastName}
                onChange={handleChange}
                placeholder="Nom"
            />
            <input 
                name="email"
                type="email"
                value={formData.email}
                onChange={handleChange}
                placeholder="Email"
            />
            <input 
                name="phone"
                type="tel"
                value={formData.phone}
                onChange={handleChange}
                placeholder="Telephone"
            />
        </form>
    );
}

Cette approche centralisee offre plusieurs avantages :
- Un seul point de vérité pour toutes les données
- Handler unifie et reutilisable
- Structure prête pour l'envoi vers une API
- Debugging simplifié avec un seul objet à inspecter

Avec cette syntaxe de base maîtrisée, vous êtes prêts à aborder les stratégies
avancées de gestion d'état, qui vous permettront de gérer des formulaires
de plus en plus complexes.

4. GESTION D'ÉTAT - Stratégies pour des formulaires évolutifs
===================================================

La gestion d'état constitue le cœur de vos formulaires React. Une stratégie
mal pensee peut rapidement transformer un formulaire simple en cauchemar de
maintenance. Cette section vous donne les outils pour evoluer sereinement.

A) STRATÉGIES D'ÉTAT - Choisir la bonne architecture dès le départ

Le choix de votre structure d'état détermine toute l'évolution future de
votre formulaire. Une decision eclairee des le depart vous evitera de
nombreuses reecritures.

MAUVAISE PRATIQUE - Trop de useState séparés :
function BadStateManagement() {
    const [firstName, setFirstName] = useState('');
    const [lastName, setLastName] = useState('');
    const [email, setEmail] = useState('');
    const [phone, setPhone] = useState('');
    // ... 20 autres états de plus !
}

Cette approche devient rapidement ingerabie et multiplie inutilement
les re-rendus du composant.

BONNE PRATIQUE - État groupé logiquement :
function GoodStateManagement() {
    const [userInfo, setUserInfo] = useState({
        firstName: '',
        lastName: '',
        email: '',
        phone: ''
    });
    
    const [shippingInfo, setShippingInfo] = useState({
        address: '',
        city: '',
        zipCode: '',
        country: ''
    });
}

Cette organisation reflete la logique metier et facilite les operations
globales sur chaque groupe de données.

    
B) PATTERNS DE MISE À JOUR - Techniques adaptées à chaque situation

Selon la structure de vos donnees, vous aurez besoin de differentes
techniques de mise à jour. Chaque pattern résout un problème spécifique.
Voyons des exemples concrets pour bien comprendre.

EXEMPLE DE STRUCTURE COMPLEXE :
Imaginons un formulaire d'inscription avec cette structure :

const [form, setForm] = useState({
    username: '',                    // Champ simple niveau 1
    email: '',                      // Champ simple niveau 1
    preferences: {                  // Objet imbrique niveau 2
        newsletter: false,
        notifications: true,
        theme: 'light'
    },
    address: {                      // Autre objet imbrique niveau 2
        street: '',
        city: '',
        zipCode: ''
    }
});

PATTERN 1 - MISE A JOUR SIMPLE : Pour les champs de premier niveau
Utilise quand : L'utilisateur tape dans username ou email

const updateSimpleField = (e) => {
    const { name, value } = e.target;
    console.log("Je mets a jour le champ :", name, "avec la valeur :", value);
    
    setForm(prev => ({
        ...prev,        // Je garde tout l'objet existant
        [name]: value   // Je change seulement le champ concerne
    }));
};

EXEMPLE CONCRET : Si l'utilisateur tape "Jean" dans le champ username :
- name = "username" 
- value = "Jean"
- Resultat : form.username devient "Jean", tout le reste inchange

PATTERN 2 - MISE A JOUR OBJETS IMBRIQUES : Pour preferences ou address
Utilise quand : L'utilisateur coche une case dans les preferences

const updateNestedField = (e) => {
    const { name, checked } = e.target;
    console.log("Je mets a jour", name, "dans preferences avec :", checked);
    
    setForm(prev => ({
        ...prev,              // Je garde username, email, address
        preferences: {        // Je reconstruis l'objet preferences
            ...prev.preferences,  // Je garde newsletter, notifications, theme existants
            [name]: checked      // Je change seulement le champ coche
        }
    }));
};

EXEMPLE CONCRET : Si l'utilisateur coche la case newsletter :
- name = "newsletter"
- checked = true
- Resultat : form.preferences.newsletter devient true
- username, email, address restent inchanges
- preferences.notifications et preferences.theme restent inchanges

PATTERN 3 - MISE À JOUR MULTIPLE : Pour modifier plusieurs champs d'un coup
Utilise quand : Vous chargez des donnees depuis une API ou resetez une section

const updateMultipleFields = (updates) => {
    console.log("Je mets a jour plusieurs champs :", updates);
    
    setForm(prev => ({
        ...prev,     // Je garde ce qui n'est pas dans updates
        ...updates   // J'ecrase avec les nouvelles valeurs
    }));
};

EXEMPLE CONCRET : Charger une adresse depuis une API :
updateMultipleFields({
    address: {
        street: '123 Rue de la Paix',
        city: 'Paris', 
        zipCode: '75001'
    }
});
Resultat : Tout l'objet address est remplace, username/email/preferences inchanges

PATTERN 4 - MISE À JOUR ADRESSE SPÉCIFIQUE : Pour les sous-objets complexes
Utilise quand : L'utilisateur tape dans un champ d'adresse

const updateAddressField = (e) => {
    const { name, value } = e.target;
    console.log("Je mets a jour", name, "dans l'adresse avec :", value);
    
    setForm(prev => ({
        ...prev,
        address: {
            ...prev.address,  // Je garde street, city, zipCode existants
            [name]: value     // Je change seulement le champ tape
        }
    }));
};

EXEMPLE CONCRET : Si l'utilisateur tape "Lyon" dans le champ city :
- name = "city"
- value = "Lyon"  
- Resultat : form.address.city devient "Lyon"
- form.address.street et form.address.zipCode inchanges
- username, email, preferences inchanges

UTILISATION DANS LE JSX - Comment associer chaque pattern :

return (
    <form>
        {/* Pattern 1 : Champs simples */}
        <input 
            name="username"
            value={form.username}
            onChange={updateSimpleField}    // Pattern simple
        />
        
        {/* Pattern 2 : Preferences imbriquees */}
        <input 
            name="newsletter"
            type="checkbox"
            checked={form.preferences.newsletter}
            onChange={updateNestedField}    // Pattern imbrique
        />
        
        {/* Pattern 4 : Champs d'adresse */}
        <input 
            name="city"
            value={form.address.city}
            onChange={updateAddressField}   // Pattern sous-objet
        />
        
        {/* Pattern 3 : Bouton pour charger des donnees */}
        <button onClick={() => updateMultipleFields({
            username: 'John',
            email: 'john@example.com'
        })}>
            Charger donnees test
        </button>
    </form>
);

RESUME DES CAS D'USAGE :
- Pattern 1 : Champ direct dans l'objet principal (username, email)
- Pattern 2 : Champ dans un sous-objet (preferences.newsletter)  
- Pattern 3 : Plusieurs champs d'un coup (chargement API, reset)
- Pattern 4 : Champ specifique dans un sous-objet (address.city)

Cette maitrise des patterns vous permet de gerer n'importe quelle structure
de données, aussi complexe soit-elle, sans jamais perdre le contrôle.

C) INITIALISATION INTELLIGENTE - Preparer vos formulaires pour tous les scenarios

Au-dela de la mise a jour, l'initialisation de vos formulaires merite une
attention particuliere. Une strategie bien pensee vous permet de gerer
élégamment les cas d'édition, les valeurs par défaut, et le chargement
de donnees externes.

function SmartInitialization({ userProfile = {} }) {
    // Initialisation avec valeurs par défaut + données existantes
    const [formData, setFormData] = useState({
        firstName: userProfile.firstName || '',
        lastName: userProfile.lastName || '',
        email: userProfile.email || '',
        phone: userProfile.phone || '',
        newsletter: userProfile.preferences?.newsletter ?? true
    });
    
    // Fonction de reinitialisation du formulaire
    const resetForm = () => {
        setFormData({
            firstName: '',
            lastName: '',
            email: '',
            phone: '',
            newsletter: true
        });
    };
    
    // Chargement de donnees externes (par exemple depuis une API)
    useEffect(() => {
        if (userProfile.id) {
            setFormData(prev => ({
                ...prev,
                ...userProfile
            }));
        }
    }, [userProfile]);

    return (
        <form>
            {/* Vos champs de formulaire */}
            <button type="button" onClick={resetForm}>
                Reinitialiser
            </button>
        </form>
    );
}

Cette approche d'initialisation intelligente couvre :
- Valeurs par défaut cohérentes
- Integration de donnees existantes (mode edition)
- Gestion des propriétés optionnelles avec l'opérateur ??
- Reinitialisation controlee
- Mise a jour automatique lors de changements externes

Maintenant que nous maitrison la gestion d'etat, explorons comment capturer
et traiter efficacement les interactions utilisateur grâce aux événements.

### A) onChange - LE CŒUR DE LA RÉACTIVITÉ

```jsx
function OnChangeExamples() {
    const [formData, setFormData] = useState({
        search: '',
        quantity: 1,
        category: ''
    });
    
    // onChange basique
    const handleBasicChange = (e) => {
        const { name, value } = e.target;
        setFormData(prev => ({ ...prev, [name]: value }));
    };
    
    // onChange avec transformation
    const handleTransformChange = (e) => {
        const { name, value } = e.target;
        let transformedValue = value;
        
        if (name === 'search') {
            transformedValue = value.toLowerCase();
        } else if (name === 'quantity') {
            transformedValue = Math.max(1, parseInt(value) || 1);
        }
        
        setFormData(prev => ({ ...prev, [name]: transformedValue }));
    };
    
    // onChange avec validation en temps réel
    const handleValidatedChange = (e) => {
        const { name, value } = e.target;
        
        // Validation côté client
        if (name === 'email' && value && !value.includes('@')) {
            console.warn('Email invalide');
        }
        
        setFormData(prev => ({ ...prev, [name]: value }));
    };
}
```

### B) onSubmit - TRAITEMENT FINAL

```jsx
function SubmitHandling() {
    const [formData, setFormData] = useState({
        username: '',
        password: ''
    });
    const [isSubmitting, setIsSubmitting] = useState(false);
    const [errors, setErrors] = useState({});
    
    const handleSubmit = async (e) => {
        e.preventDefault(); // ❗ ESSENTIEL - Empêche le rechargement
        
        setIsSubmitting(true);
        setErrors({});
        
        try {
            // Validation avant envoi
            const validationErrors = validateForm(formData);
            if (Object.keys(validationErrors).length > 0) {
                setErrors(validationErrors);
                return;
            }
            
            // Envoi des données
            const response = await fetch('/api/login', {
                method: 'POST',
                headers: { 'Content-Type': 'application/json' },
                body: JSON.stringify(formData)
            });
            
            if (response.ok) {
                console.log('Succès !');
                // Redirection ou reset du formulaire
            } else {
                setErrors({ general: 'Erreur de connexion' });
            }
            
        } catch (error) {
            setErrors({ general: 'Erreur réseau' });
        } finally {
            setIsSubmitting(false);
        }
    };
    
    return (
        <form onSubmit={handleSubmit}>
            {/* Champs du formulaire */}
            <button 
                type="submit" 
                disabled={isSubmitting}
            >
                {isSubmitting ? 'Connexion...' : 'Se connecter'}
            </button>
            
            {errors.general && (
                <p className="error">{errors.general}</p>
            )}
        </form>
    );
}
```

### C) onBlur - VALIDATION À LA PERTE DE FOCUS

```jsx
function BlurValidation() {
    const [formData, setFormData] = useState({
        email: '',
        password: ''
    });
    const [touchedFields, setTouchedFields] = useState({});
    const [errors, setErrors] = useState({});
    
    const handleBlur = (e) => {
        const { name, value } = e.target;
        
        // Marquer le champ comme "touché"
        setTouchedFields(prev => ({ ...prev, [name]: true }));
        
        // Validation spécifique
        let error = '';
        if (name === 'email' && value && !isValidEmail(value)) {
            error = 'Format email invalide';
        } else if (name === 'password' && value && value.length < 8) {
            error = 'Mot de passe trop court';
        }
        
        setErrors(prev => ({ ...prev, [name]: error }));
    };
    
    const isValidEmail = (email) => {
        return /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(email);
    };
    
    return (
        <form>
            <input 
                name="email"
                type="email"
                value={formData.email}
                onChange={handleChange}
                onBlur={handleBlur}
                className={errors.email && touchedFields.email ? 'error' : ''}
            />
            {errors.email && touchedFields.email && (
                <span className="error-message">{errors.email}</span>
            )}
        </form>
    );
}
```

```jsx
// Utilitaires de validation
const validators = {
    required: (value) => value.trim() !== '' || 'Ce champ est obligatoire',
    email: (value) => {
        if (!value) return true;
        return /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(value) || 'Email invalide';
    },
    minLength: (min) => (value) => {
        if (!value) return true;
        return value.length >= min || `Minimum ${min} caractères`;
    },
    phone: (value) => {
        if (!value) return true;
        return /^[\d\s\-\+\(\)\.]{10,}$/.test(value) || 'Numéro invalide';
    }
};

function ValidationExample() {
    const [formData, setFormData] = useState({
        firstName: '',
        email: '',
        phone: '',
        password: ''
    });
    const [errors, setErrors] = useState({});
    const [touchedFields, setTouchedFields] = useState({});
    
    // Règles de validation par champ
    const validationRules = {
        firstName: [validators.required],
        email: [validators.required, validators.email],
        phone: [validators.phone],
        password: [validators.required, validators.minLength(8)]
    };
    
    // Validation d'un champ
    const validateField = (name, value) => {
        const rules = validationRules[name] || [];
        for (let rule of rules) {
            const result = rule(value);
            if (result !== true) {
                return result; // Premier erreur trouvée
            }
        }
        return '';
    };
    
    // Validation complète du formulaire
    const validateForm = () => {
        const newErrors = {};
        Object.keys(validationRules).forEach(field => {
            const error = validateField(field, formData[field]);
            if (error) newErrors[field] = error;
        });
        setErrors(newErrors);
        return Object.keys(newErrors).length === 0;
    };
    
    const handleChange = (e) => {
        const { name, value } = e.target;
        setFormData(prev => ({ ...prev, [name]: value }));
        
        // Validation en temps réel si le champ a été touché
        if (touchedFields[name]) {
            const error = validateField(name, value);
            setErrors(prev => ({ ...prev, [name]: error }));
        }
    };
    
    const handleBlur = (e) => {
        const { name, value } = e.target;
        setTouchedFields(prev => ({ ...prev, [name]: true }));
        
        const error = validateField(name, value);
        setErrors(prev => ({ ...prev, [name]: error }));
    };
    
    const handleSubmit = (e) => {
        e.preventDefault();
        
        // Marquer tous les champs comme touchés
        const allTouched = Object.keys(validationRules).reduce((acc, field) => {
            acc[field] = true;
            return acc;
        }, {});
        setTouchedFields(allTouched);
        
        // Validation complète
        if (validateForm()) {
            console.log('Formulaire valide !', formData);
            // Traitement...
        }
    };
}
```

```jsx
function VisualFeedback() {

        if (/[A-Z]/.test(pwd)) strength++;
        if (/\d/.test(pwd)) strength++;
        if (/[^a-zA-Z\d]/.test(pwd)) strength++;

        const colors = ['#ff4444', '#ff8800', '#ffdd00', '#88dd00', '#00dd00'];
        
        return {
            level: levels[strength] || 'Aucun',
            color: colors[strength] || '#cccccc',
            score: strength
        };
    };
    
    const passwordStrength = getPasswordStrength(password);
    const passwordsMatch = password === confirmPassword && password !== '';
    
    return (

                    value={password}

                                }}

                            {passwordStrength.level}

                    </div>
                )}

        </form>
    );
```

```tsx
// Définition des types d'interface
interface UserData {
    firstName: string;
    lastName: string;
    email: string;
    phone?: string;
    birthDate?: Date;
}

interface ValidationRule {
    (value: string): string | true;
}

interface ValidationRules {
    [fieldName: string]: ValidationRule[];
}

interface UserFormProps {
    initialData?: Partial<UserData>;
    onSubmit: (data: UserData) => Promise<void>;
    isLoading?: boolean;
    validationRules?: ValidationRules;
    showOptionalFields?: boolean;
    onFieldChange?: (fieldName: keyof UserData, value: string) => void;
}

// Composant avec types stricts
const UserForm: React.FC<UserFormProps> = ({
    initialData = {},
    onSubmit,
    isLoading = false,
    validationRules = {},
    showOptionalFields = false,
    onFieldChange
}) => {
    const [formData, setFormData] = useState<UserData>({
        firstName: initialData.firstName || '',
        lastName: initialData.lastName || '',
        email: initialData.email || '',
        phone: initialData.phone || '',
        birthDate: initialData.birthDate
    });
    
    const [errors, setErrors] = useState<Partial<Record<keyof UserData, string>>>({});
    
    const handleChange = (e: React.ChangeEvent<HTMLInputElement>) => {
        const { name, value } = e.target;
        const fieldName = name as keyof UserData;
        
        setFormData(prev => ({ ...prev, [fieldName]: value }));
        
        // Callback optionnel
        if (onFieldChange) {
            onFieldChange(fieldName, value);
        }
        
        // Validation si définie
        if (validationRules[fieldName]) {
            const rules = validationRules[fieldName];
            for (let rule of rules) {
                const result = rule(value);
                if (result !== true) {
                    setErrors(prev => ({ ...prev, [fieldName]: result }));
                    break;
                }
            }
        }
    };
    
    const handleSubmit = async (e: React.FormEvent) => {
        e.preventDefault();
        
        try {
            await onSubmit(formData);
        } catch (error) {
            console.error('Erreur de soumission:', error);
        }
    };
    
    return (
        <form onSubmit={handleSubmit}>
            <input 
                name="firstName"
                type="text"
                value={formData.firstName}
                onChange={handleChange}
                disabled={isLoading}
                aria-invalid={!!errors.firstName}
                aria-describedby={errors.firstName ? "firstName-error" : undefined}
            />
            
            {errors.firstName && (
                <span id="firstName-error" role="alert">
                    {errors.firstName}
                </span>
            )}
            
            {showOptionalFields && (
                <input 
                    name="phone"
                    type="tel"
                    value={formData.phone || ''}
                    onChange={handleChange}
                    disabled={isLoading}
                />
            )}
            
            <button type="submit" disabled={isLoading}>
                {isLoading ? 'Envoi...' : 'Valider'}
            </button>
        </form>
    );
};

export default UserForm;
```

```tsx
function useValidatedProps<T>(props: T, validator: (props: T) => string[]): T {
    useEffect(() => {
        if (process.env.NODE_ENV === 'development') {
            const errors = validator(props);
            if (errors.length > 0) {
                console.warn('Props validation errors:', errors);
            }
        }
    }, [props, validator]);
    
    return props;
}
```
