OBJECTIF  :
Nous allons maintenant gérer un panier d'achat ! Pour cela, nous devons créer un composant Cart qui va afficher les produits ajoutés et permettre de gérer les quantités ainsi que le montant total des achats.

PROBLÉMATIQUE : COMMUNICATION ENTRE COMPOSANTS FRÈRES
Nos composants ShoppingList et Cart seront  des "composants frères" :
• Ils sont au même niveau hiérarchique
• Ils ont besoin de partager les MÊMES données (contenu du panier)
• En React, les composants frères ne peuvent PAS communiquer directement

SCHÉMA DU PROBLÈME
App
├-- ShoppingList (veut ajouter au panier)
 └--- Cart (veut afficher le panier)
    ↑
Comment faire communiquer ces deux composants ?

PLUSIEURS SOLUTIONS EN REACT
OPTION 1 : LIFTING STATE UP (REMONTÉE D'ÉTAT)
Principe : Remonter l'état partagé vers le parent commun le plus proche
• Avantages : Simple, prévisible, contrôle total
• Inconvénients : Props drilling si hiérarchie profonde, il faut faire passer les données étape par étape
• Usage : Idéal pour applications moyennes

OPTION 2 : CONTEXT API
• Principe : État global accessible partout dans l'arbre
•  Avantages : Évite le props drilling
• Inconvénients : Plus complexe, peut créer des re-rendus inutiles
• Usage : État vraiment global (thème, utilisateur connecté)


OPTION 3 : REDUX/ZUSTAND
Principe : Store externe à React
• Avantages : Très puissant, debugging avancé
• Inconvénients : plus difficile à mettre en place
• Usage : Applications complexes avec beaucoup d'état

SOLUTION CHOISIE : LIFTING STATE UP

✓ Parfaite pour notre cas d'usage (e-commerce simple)
✓ Concepts React fondamentaux 
✓ Code prévisible et facile à déboguer

ÉTAPE 1 : MODIFICATION DE App.jsx
---------------------------------
Objectif : Créer l'état centralisé et les fonctions de gestion

CODE À AJOUTER :
```javascript
import { useState } from 'react';
import ShoppingList from './Components/ShoppingList';
import Cart from './Components/Cart';

function App() {
    // État centralisé du panier
    const [cart, setCart] = useState([]);
    
    // Fonction pour ajouter un produit au panier
    const addToCart = (sneaker) => {
        setCart(prevCart => {
            // Vérifier si le produit existe déjà
            const existingItem = prevCart.find(item => item.id === sneaker.id);
            
            if (existingItem) {
                // Augmenter la quantité
                return prevCart.map(item =>
                    item.id === sneaker.id 
                        ? { ...item, quantity: item.quantity + 1 }
                        : item
                );
            } else {
                // Ajouter nouveau produit avec quantité 1
                return [...prevCart, { ...sneaker, quantity: 1 }];
            }
        });
    };
    
    return (
        <div className="App">
            <ShoppingList onAddToCart={addToCart} />
            <Cart cartItems={cart} />
        </div>
    );
}

export default App;
```

CONCEPTS CLÉS EXPLIQUÉS :
- useState([]) : Initialise un panier vide
- addToCart : Fonction qui gère la logique d'ajout
- Immutabilité : Utilisation de spread operator et map
- Props : Transmission des données et fonctions aux enfants

```javascript
import React from 'react';

function Cart({ cartItems }) {

            </div>
            
            <div className="cart-items">
                {cartItems.map(item => (

                        <img src={item.image} alt={item.nom} className="cart-item-image" />
                        <div className="cart-item-details">
                            <h4>{item.nom}</h4>
                            <p>{item.marque}</p>
                            <p>Quantité: {item.quantity}</p>
                            <p className="cart-item-price">{item.prix * item.quantity}€</p>
                        </div>
                    </div>
                ))}
            </div>
            
            <div className="cart-summary">
                <h3>Total: {totalPrice}€</h3>
            </div>
        </div>
    );
}

export default Cart;
```

CONCEPTS CLÉS :

OPTIMISATION AVEC LES HOOKS À DÉPENDANCES
=============================================

INTRODUCTION AUX HOOKS AVEC DÉPENDANCES
----------------------------------------
Maintenant que notre panier fonctionne, nous allons l'optimiser avec des hooks avancés :
- useEffect : Pour la persistance des données
- useCallback : Pour optimiser les fonctions
- useMemo : Pour optimiser les calculs

Ces hooks utilisent un système de "dépendances" qui détermine QUAND ils doivent se ré-exécuter.

PRINCIPE DES DÉPENDANCES
------------------------
Tous ces hooks acceptent un tableau de dépendances en second paramètre :

```javascript
useEffect(() => { /* effet */ }, [dependency1, dependency2]);
useCallback(() => { /* fonction */ }, [dependency1]);
useMemo(() => { /* calcul */ }, [dependency1, dependency2]);
```

RÈGLES DES DÉPENDANCES :
- [] (tableau vide) = exécute UNE SEULE FOIS
- [a, b] = exécute si 'a' OU 'b' change
- pas de tableau = exécute à CHAQUE rendu (généralement évité)

ÉTAPE 5 : PERSISTANCE AVEC useEffect
=====================================

OBJECTIF : SAUVEGARDER LE PANIER AUTOMATIQUEMENT
------------------------------------------------
Chaque fois que le panier change, on veut le sauvegarder dans localStorage.

CODE À AJOUTER DANS APP.JSX :
```javascript
import { useState, useEffect } from 'react';

function App() {
    // État centralisé du panier
    const [cart, setCart] = useState([]);
    
    // EFFET : Charger le panier au démarrage
    useEffect(() => {
        const savedCart = localStorage.getItem('sneakers-cart');
        if (savedCart) {
            setCart(JSON.parse(savedCart));
        }
    }, []); // [] = s'exécute UNE SEULE FOIS au montage
    
    // EFFET : Sauvegarder à chaque changement du panier
    useEffect(() => {
        localStorage.setItem('sneakers-cart', JSON.stringify(cart));
    }, [cart]); // [cart] = s'exécute quand 'cart' change
    
    // ... reste du code
}
```

EXPLICATION DES DÉPENDANCES :
- Premier useEffect ([]) : Une seule fois = chargement initial
- Deuxième useEffect ([cart]) : À chaque changement du panier = sauvegarde

ÉTAPE 6 : OPTIMISATION AVEC useCallback
========================================

PROBLÈME : FONCTIONS RECRÉÉES À CHAQUE RENDU
--------------------------------------------
Actuellement, addToCart est recréée à chaque rendu d'App.
Cela peut causer des re-rendus inutiles des composants enfants.

SOLUTION : MÉMORISER LA FONCTION
--------------------------------
```javascript
import { useState, useEffect, useCallback } from 'react';

function App() {
    const [cart, setCart] = useState([]);
    
    // Fonction optimisée avec useCallback
    const addToCart = useCallback((sneaker) => {
        setCart(prevCart => {
            const existingItem = prevCart.find(item => item.id === sneaker.id);
            
            if (existingItem) {
                return prevCart.map(item =>
                    item.id === sneaker.id 
                        ? { ...item, quantity: item.quantity + 1 }
                        : item
                );
            } else {
                return [...prevCart, { ...sneaker, quantity: 1 }];
            }
        });
    }, []); // [] = fonction stable, ne change jamais
    
    // Fonction pour supprimer (nouvelle fonctionnalité)
    const removeFromCart = useCallback((sneakerId) => {
        setCart(prevCart => prevCart.filter(item => item.id !== sneakerId));
    }, []); // [] = fonction stable
    
    // ... useEffect comme avant
}
```

POURQUOI useCallback ICI ?
- addToCart n'a pas de dépendances externes
- Elle utilise la fonction de mise à jour avec prevCart
- Elle reste stable = évite les re-rendus inutiles

ÉTAPE 7 : OPTIMISATION AVEC useMemo
===================================

PROBLÈME : CALCULS RÉPÉTÉS À CHAQUE RENDU
-----------------------------------------
Dans Cart.jsx, les totaux sont recalculés à chaque rendu, même si le panier n'a pas changé.

SOLUTION : MÉMORISER LES CALCULS
-------------------------------
```javascript
import React, { useMemo } from 'react';

function Cart({ cartItems }) {
    // Calculs optimisés avec useMemo
    const cartStats = useMemo(() => {
        const totalItems = cartItems.reduce((sum, item) => sum + item.quantity, 0);
        const totalPrice = cartItems.reduce((sum, item) => sum + (item.prix * item.quantity), 0);
        
        return { totalItems, totalPrice };
    }, [cartItems]); // [cartItems] = recalcule si le panier change
    
    // Calcul des articles groupés par marque
    const itemsByBrand = useMemo(() => {
        return cartItems.reduce((acc, item) => {
            acc[item.marque] = (acc[item.marque] || 0) + item.quantity;
            return acc;
        }, {});
    }, [cartItems]); // [cartItems] = recalcule si le panier change
    
    if (cartItems.length === 0) {
        return (
            <div className="cart">
                <h2>Mon Panier</h2>
                <p className="empty-cart">Votre panier est vide</p>
            </div>
        );
    }
    
    return (
        <div className="cart">
            <div className="cart-header">
                <h2>Mon Panier ({cartStats.totalItems})</h2>
            </div>
            
            {/* Affichage par marque */}
            <div className="cart-brands">
                <h3>Par marque :</h3>
                {Object.entries(itemsByBrand).map(([brand, count]) => (
                    <span key={brand} className="brand-count">
                        {brand}: {count}
                    </span>
                ))}
            </div>
            
            <div className="cart-items">
                {cartItems.map(item => (
                    <div key={item.id} className="cart-item">
                        <img src={item.image} alt={item.nom} className="cart-item-image" />
                        <div className="cart-item-details">
                            <h4>{item.nom}</h4>
                            <p>{item.marque}</p>
                            <p>Quantité: {item.quantity}</p>
                            <p className="cart-item-price">{item.prix * item.quantity}€</p>
                        </div>
                    </div>
                ))}
            </div>
            
            <div className="cart-summary">
                <h3>Total: {cartStats.totalPrice}€</h3>
            </div>
        </div>
    );
}
```

POURQUOI useMemo ICI ?
- Calculs potentiellement coûteux (reduce sur gros paniers)
- Résultats dépendent uniquement de cartItems
- Évite les recalculs inutiles

COMPARAISON DES HOOKS À DÉPENDANCES
===================================

| Hook        | Usage                    | Retourne           | Dépendances typiques |
|-------------|--------------------------|--------------------|--------------------|
| useEffect   | Effets de bord           | Rien (void)        | [state, props]     |
| useCallback | Mémoriser des fonctions  | Fonction mémorisée | [state utilisé]    |
| useMemo     | Mémoriser des calculs    | Valeur mémorisée   | [données sources]  |

EXEMPLES CONCRETS DANS NOTRE PANIER
===================================

```javascript
// useEffect - Sauvegarde automatique
useEffect(() => {
    localStorage.setItem('cart', JSON.stringify(cart));
}, [cart]); // Se relance si 'cart' change

// useCallback - Fonction stable
const addToCart = useCallback((sneaker) => {
    setCart(prev => [...prev, sneaker]);
}, []); // Fonction ne change jamais

// useMemo - Calcul optimisé
const total = useMemo(() => {
    return cart.reduce((sum, item) => sum + item.price, 0);
}, [cart]); // Recalcule si 'cart' change
```

BONNES PRATIQUES POUR LES DÉPENDANCES
=====================================

❌ ERREURS COURANTES :
```javascript
// Oublier une dépendance
useEffect(() => {
    setCount(count + step); // 'step' manque dans les dépendances
}, [count]); // ❌ ERREUR: 'step' devrait être inclus

// Dépendance inutile
useMemo(() => {
    return cart.length;
}, [cart, user]); // ❌ 'user' n'est pas utilisé
```

✅ BONNES PRATIQUES :
```javascript
// Toutes les dépendances incluses
useEffect(() => {
    setCount(count + step);
}, [count, step]); // ✅ Toutes les variables utilisées

// Fonction de mise à jour pour éviter les dépendances
useEffect(() => {
    setCount(prevCount => prevCount + step);
}, [step]); // ✅ Plus de dépendance sur 'count'
```

ORDRE PÉDAGOGIQUE RECOMMANDÉ
============================

1. **useState** - État local (avis, panier)
2. **useEffect** - Effets simples (sauvegarde)
3. **useCallback** - Fonctions optimisées
4. **useMemo** - Calculs optimisés
5. **Règles des dépendances** - Bonnes pratiques

QUAND UTILISER CES OPTIMISATIONS ?
==================================

**useEffect** : Toujours nécessaire pour les effets de bord
**useCallback** : Quand la fonction est passée à de nombreux enfants
**useMemo** : Quand les calculs sont visiblement coûteux

⚠️ **ATTENTION** : Ne pas sur-optimiser ! 
Ces hooks ont un coût. Ne les utilisez que quand c'est vraiment bénéfique.
