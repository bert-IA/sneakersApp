GUIDE RAPIDE : FORMULAIRES REACT POUR LIVRAISON
==============================================

OBJECTIF : Maîtriser rapidement les outils React nécessaires pour créer 
un formulaire de livraison professionnel avec validation et gestion d'état.

===============================================================================

1. PATTERN DE BASE - Controlled Components
==========================================

CE QU'ON VA CRÉER : Un formulaire React qui capture toutes les informations 
de livraison avec une gestion d'état centralisée. Nous allons apprendre pourquoi 
les "Controlled Components" sont la base de tous les formulaires React professionnels.

Concept clé : Un "Controlled Component" signifie que React garde la valeur de 
chaque input dans son état (state) et la synchronise en permanence avec l'affichage.

Le Template Standard
--------------------

ÉTAT CENTRALISÉ : Au lieu de créer une variable useState pour chaque champ 
(firstName, lastName, email...), on groupe TOUT dans un seul objet. Cela 
simplifie la gestion et prépare l'envoi vers une API.

import { useState } from 'react';

function ShippingForm() {
    const [formData, setFormData] = useState({
        firstName: '',
        lastName: '',
        email: '',
        phone: '',
        address: '',
        city: '',
        postalCode: '',
        country: 'France',
        deliveryOptions: 'standard',
        newsletter: false
    });

HANDLER UNIVERSEL : Une "handler" est une fonction qui réagit aux événements 
(comme taper dans un champ). Au lieu d'écrire une fonction différente pour 
chaque input, cette fonction unique fonctionne pour TOUS les types d'inputs.

    const handleChange = (e) => {
        const { name, value, type, checked } = e.target;
        setFormData(prev => ({
            ...prev,
            [name]: type === 'checkbox' ? checked : value
        }));
    };

    return (
        <form onSubmit={handleSubmit}>
            {/* Vos inputs ici */}
        </form>
    );
}

MISE À JOUR IMMUTABLE : Cette technique crée un nouvel objet en gardant tout 
l'ancien contenu (...prev) et en modifiant seulement le champ concerné. C'est 
essentiel pour que React détecte les changements et mette à jour l'affichage.

POURQUOI CETTE APPROCHE : Contrairement au HTML classique où le navigateur stocke 
les valeurs, ici React contrôle tout. Cela permet la validation en temps réel, 
la logique conditionnelle, et une expérience utilisateur plus fluide.

AVANTAGES de ce pattern :
- Un seul handler pour tous les champs = moins de code répétitif
- Structure prête pour l'API = envoi direct avec JSON.stringify(formData)
- État toujours synchronisé = pas de décalage entre affichage et données
- Facilite les validations = accès immédiat à toutes les valeurs

PROCHAINE ÉTAPE : Maintenant qu'on maîtrise le concept de base, voyons comment 
l'appliquer concrètement aux différents types d'inputs que vous utiliserez dans 
votre formulaire de livraison.

2. TYPES D'INPUTS ESSENTIELS
============================

CE QU'ON VA APPRENDRE : Comment utiliser le même pattern handleChange avec tous 
les types d'inputs nécessaires pour un formulaire de livraison complet. Chaque 
type a ses spécificités mais suit la même logique de base.

Input Texte Standard
--------------------

<input 
    name="firstName"
    type="text"
    value={formData.firstName}
    onChange={handleChange}
    placeholder="Prénom *"
    required
/>

Input Email avec Validation
---------------------------

<input 
    name="email"
    type="email"
    value={formData.email}
    onChange={handleChange}
    placeholder="Email *"
    required
    pattern="[a-z0-9._%+-]+@[a-z0-9.-]+\.[a-z]{2,}$"
/>

ATTRIBUT PATTERN : Ce pattern HTML5 valide automatiquement le format email côté 
navigateur. React récupère cette validation native pour améliorer l'expérience.

Select (Liste déroulante)
-------------------------

<select 
    name="country"
    value={formData.country}
    onChange={handleChange}
    required
>
    <option value="">Choisir un pays</option>
    <option value="France">France</option>
    <option value="Belgique">Belgique</option>
    <option value="Suisse">Suisse</option>
</select>

PARTICULARITÉ DES SELECT : La valeur sélectionnée est contrôlée par l'attribut 
value du select, pas par selected sur les options. Notre handleChange fonctionne 
parfaitement sans modification.

Radio Buttons (Options exclusives)
-----------------------------------

<div className="radio-group">
    <label>
        <input 
            name="deliveryOptions"
            type="radio"
            value="standard"
            checked={formData.deliveryOptions === 'standard'}
            onChange={handleChange}
        />
        Livraison standard (3-5 jours) - Gratuit
    </label>
    
    <label>
        <input 
            name="deliveryOptions"
            type="radio"
            value="express"
            checked={formData.deliveryOptions === 'express'}
            onChange={handleChange}
        />
        Livraison express (24h) - 9,99€
    </label>
</div>

RADIO BUTTONS PARTICULIERS : Chaque radio a la même propriété "name" mais des 
"value" différentes. L'attribut "checked" compare la valeur de l'état avec la 
valeur de ce radio spécifique. Notre handleChange met automatiquement à jour 
la bonne propriété.

Checkbox
--------

<label>
    <input 
        name="newsletter"
        type="checkbox"
        checked={formData.newsletter}
        onChange={handleChange}
    />
    S'abonner à notre newsletter
</label>

CHECKBOX SPÉCIAL : Contrairement aux autres inputs, les checkbox utilisent 
"checked" au lieu de "value". Notre handleChange gère déjà ce cas avec la 
condition "type === 'checkbox' ? checked : value".

RÉCAPITULATIF INPUTS : Tous ces types d'inputs différents utilisent exactement 
le même handleChange ! C'est la puissance de cette approche : une seule fonction 
pour gérer toute la complexité.

3. GESTION DE LA SOUMISSION
===========================

CE QU'ON VA APPRENDRE : Comment traiter l'envoi du formulaire de manière 
professionnelle avec validation, gestion d'erreurs, et états de chargement. 
C'est ce qui transforme un simple formulaire en expérience utilisateur fluide.

const [isSubmitting, setIsSubmitting] = useState(false);
const [errors, setErrors] = useState({});
const [success, setSuccess] = useState(false);

ÉTATS SUPPLÉMENTAIRES NÉCESSAIRES : En plus de formData, nous devons gérer 
l'état de chargement (pour désactiver le bouton), les erreurs de validation, 
et le succès de l'opération.

const handleSubmit = async (e) => {
    e.preventDefault();

    setIsSubmitting(true);
    setErrors({});

    try {
        const validationErrors = validateForm(formData);
        if (Object.keys(validationErrors).length > 0) {
            setErrors(validationErrors);
            return;
        }

        const response = await fetch('/api/shipping', {
            method: 'POST',
            headers: { 'Content-Type': 'application/json' },
            body: JSON.stringify(formData)
        });

        if (response.ok) {
            setSuccess(true);
        } else {
            setErrors({ general: 'Erreur lors de l\'envoi' });
        }

    } catch (error) {
        setErrors({ general: 'Erreur de connexion' });
    } finally {
        setIsSubmitting(false);
    }
};

ÉTAPES DE LA SOUMISSION :
1. preventDefault() empêche le rechargement de page (comportement HTML par défaut)
2. On active l'état de chargement et on nettoie les anciennes erreurs
3. Validation côté client avant d'envoyer quoi que ce soit au serveur
4. Si validation OK, envoi des données vers l'API
5. Gestion des réponses : succès ou erreur
6. finally s'exécute toujours pour désactiver le chargement

GESTION DES ERREURS : Nous prévoyons deux types d'erreurs : les erreurs de 
validation (champs manquants/invalides) et les erreurs techniques (réseau, serveur).

4. VALIDATION SIMPLE ET EFFICACE
================================

CE QU'ON VA APPRENDRE : Comment créer une fonction de validation réutilisable 
qui vérifie tous les champs obligatoires et leurs formats. Cette validation 
côté client améliore l'expérience utilisateur en détectant les erreurs immédiatement.

const validateForm = (data) => {
    const errors = {};

    if (!data.firstName.trim()) errors.firstName = 'Prénom requis';
    if (!data.lastName.trim()) errors.lastName = 'Nom requis';
    if (!data.email.trim()) errors.email = 'Email requis';
    if (!data.address.trim()) errors.address = 'Adresse requise';
    if (!data.city.trim()) errors.city = 'Ville requise';
    if (!data.postalCode.trim()) errors.postalCode = 'Code postal requis';

    const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
    if (data.email && !emailRegex.test(data.email)) {
        errors.email = 'Format email invalide';
    }

    const postalRegex = /^[0-9]{5}$/;
    if (data.postalCode && !postalRegex.test(data.postalCode)) {
        errors.postalCode = 'Code postal invalide (5 chiffres)';
    }

    if (data.phone && data.phone.length < 10) {
        errors.phone = 'Numéro de téléphone invalide';
    }

    return errors;
};

LOGIQUE DE VALIDATION : Cette fonction parcourt toutes les données et retourne 
un objet contenant les erreurs trouvées. Si l'objet est vide (pas d'erreurs), 
la validation est réussie.

MÉTHODE trim() : Elle enlève les espaces au début et à la fin. Un champ rempli 
seulement d'espaces est considéré comme vide.

EXPRESSIONS RÉGULIÈRES (REGEX) : Ces patterns permettent de vérifier le format 
des données. L'emailRegex vérifie la structure @domain.extension.

Affichage des erreurs
---------------------

const ErrorMessage = ({ error }) => (
    error ? <span className="error-message">{error}</span> : null
);

Usage dans le JSX :

<input name="firstName" />
<ErrorMessage error={errors.firstName} />

COMPOSANT ERREUR RÉUTILISABLE : Ce petit composant affiche le message d'erreur 
seulement s'il en existe un. Sinon, il ne rend rien (null).

UTILISATION : Placez ce composant juste après chaque input. Il s'affichera 
automatiquement si la validation détecte une erreur sur ce champ.

5. COMPOSANT COMPLET PRÊT À L'EMPLOI
====================================

CE QU'ON VA VOIR : Le composant final qui combine tous les concepts précédents. 
Ce code est directement utilisable et intègre la gestion d'état, la validation, 
les erreurs, et une expérience utilisateur professionnelle.

import { useState } from 'react';

function ShippingForm({ onShippingComplete }) {
    const [formData, setFormData] = useState({
        firstName: '',
        lastName: '',
        email: '',
        phone: '',
        address: '',
        city: '',
        postalCode: '',
        country: 'France',
        deliveryOptions: 'standard',
        newsletter: false
    });

    const [isSubmitting, setIsSubmitting] = useState(false);
    const [errors, setErrors] = useState({});
    const [success, setSuccess] = useState(false);

    const handleChange = (e) => {
        const { name, value, type, checked } = e.target;
        setFormData(prev => ({
            ...prev,
            [name]: type === 'checkbox' ? checked : value
        }));
        
        if (errors[name]) {
            setErrors(prev => ({ ...prev, [name]: '' }));
        }
    };

    const validateForm = (data) => {
        const errors = {};
        
        if (!data.firstName.trim()) errors.firstName = 'Prénom requis';
        if (!data.lastName.trim()) errors.lastName = 'Nom requis';
        if (!data.email.trim()) errors.email = 'Email requis';
        if (!data.address.trim()) errors.address = 'Adresse requise';
        if (!data.city.trim()) errors.city = 'Ville requise';
        if (!data.postalCode.trim()) errors.postalCode = 'Code postal requis';
        
        const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
        if (data.email && !emailRegex.test(data.email)) {
            errors.email = 'Format email invalide';
        }
        
        const postalRegex = /^[0-9]{5}$/;
        if (data.postalCode && !postalRegex.test(data.postalCode)) {
            errors.postalCode = 'Code postal invalide';
        }
        
        return errors;
    };

    const handleSubmit = async (e) => {
        e.preventDefault();
        
        setIsSubmitting(true);
        setErrors({});
        
        try {
            const validationErrors = validateForm(formData);
            if (Object.keys(validationErrors).length > 0) {
                setErrors(validationErrors);
                return;
            }
            
            await new Promise(resolve => setTimeout(resolve, 1000));
            
            setSuccess(true);
            onShippingComplete?.(formData);
            
        } catch (error) {
            setErrors({ general: 'Erreur lors de l\'envoi' });
        } finally {
            setIsSubmitting(false);
        }
    };

    if (success) {
        return (
            <div className="success-message">
                <h2>Commande confirmée !</h2>
                <p>Vos informations de livraison ont été enregistrées.</p>
            </div>
        );
    }

    return (
        <div className="shipping-form">
            <h2>Informations de livraison</h2>
            
            <form onSubmit={handleSubmit}>
                <fieldset>
                    <legend>Informations personnelles</legend>
                    
                    <div className="form-group">
                        <label htmlFor="firstName">Prénom *</label>
                        <input 
                            id="firstName"
                            name="firstName"
                            type="text"
                            value={formData.firstName}
                            onChange={handleChange}
                            disabled={isSubmitting}
                        />
                        {errors.firstName && 
                            <span className="error-text">{errors.firstName}</span>
                        }
                    </div>
                    
                    <div className="form-group">
                        <label htmlFor="lastName">Nom *</label>
                        <input 
                            id="lastName"
                            name="lastName"
                            type="text"
                            value={formData.lastName}
                            onChange={handleChange}
                            disabled={isSubmitting}
                        />
                        {errors.lastName && 
                            <span className="error-text">{errors.lastName}</span>
                        }
                    </div>
                    
                    <div className="form-group">
                        <label htmlFor="email">Email *</label>
                        <input 
                            id="email"
                            name="email"
                            type="email"
                            value={formData.email}
                            onChange={handleChange}
                            disabled={isSubmitting}
                        />
                        {errors.email && 
                            <span className="error-text">{errors.email}</span>
                        }
                    </div>
                </fieldset>

                <fieldset>
                    <legend>Adresse de livraison</legend>
                    
                    <div className="form-group">
                        <label htmlFor="address">Adresse *</label>
                        <input 
                            id="address"
                            name="address"
                            type="text"
                            value={formData.address}
                            onChange={handleChange}
                            disabled={isSubmitting}
                        />
                        {errors.address && 
                            <span className="error-text">{errors.address}</span>
                        }
                    </div>
                    
                    <div className="form-group">
                        <label htmlFor="city">Ville *</label>
                        <input 
                            id="city"
                            name="city"
                            type="text"
                            value={formData.city}
                            onChange={handleChange}
                            disabled={isSubmitting}
                        />
                        {errors.city && 
                            <span className="error-text">{errors.city}</span>
                        }
                    </div>
                    
                    <div className="form-group">
                        <label htmlFor="postalCode">Code postal *</label>
                        <input 
                            id="postalCode"
                            name="postalCode"
                            type="text"
                            value={formData.postalCode}
                            onChange={handleChange}
                            maxLength={5}
                            disabled={isSubmitting}
                        />
                        {errors.postalCode && 
                            <span className="error-text">{errors.postalCode}</span>
                        }
                    </div>
                </fieldset>

                <fieldset>
                    <legend>Options de livraison</legend>
                    
                    <div className="radio-group">
                        <label>
                            <input 
                                name="deliveryOptions"
                                type="radio"
                                value="standard"
                                checked={formData.deliveryOptions === 'standard'}
                                onChange={handleChange}
                                disabled={isSubmitting}
                            />
                            Livraison standard (3-5 jours) - Gratuit
                        </label>
                        
                        <label>
                            <input 
                                name="deliveryOptions"
                                type="radio"
                                value="express"
                                checked={formData.deliveryOptions === 'express'}
                                onChange={handleChange}
                                disabled={isSubmitting}
                            />
                            Livraison express (24h) - 9,99€
                        </label>
                    </div>
                </fieldset>

                <fieldset>
                    <legend>Préférences</legend>
                    
                    <label>
                        <input 
                            name="newsletter"
                            type="checkbox"
                            checked={formData.newsletter}
                            onChange={handleChange}
                            disabled={isSubmitting}
                        />
                        S'abonner à notre newsletter
                    </label>
                </fieldset>

                {errors.general && (
                    <div className="error-banner">
                        {errors.general}
                    </div>
                )}

                <button 
                    type="submit" 
                    disabled={isSubmitting}
                >
                    {isSubmitting ? 'Traitement...' : 'Confirmer la commande'}
                </button>
            </form>
        </div>
    );
}

export default ShippingForm;

POINTS CLÉS DE CE COMPOSANT :
- Un seul useState avec objet centralisé pour toutes les données
- Un seul handleChange pour tous les inputs
- Validation côté client avant envoi
- Gestion d'état de chargement avec désactivation des champs
- Messages d'erreur spécifiques par champ
- Nettoyage automatique des erreurs quand l'utilisateur corrige
- Écran de succès après validation

ÉTAPE 2 : Modifier handleChange pour les propriétés imbriquées

const handleChange = (e) => {
    const { name, value, type, checked } = e.target;
    
    // Gestion des services additionnels (propriétés imbriquées)
    if (name.startsWith('additionalServices.')) {
        const serviceName = name.split('.')[1];
        setFormData(prev => ({
            ...prev,
            additionalServices: {
                ...prev.additionalServices,
                [serviceName]: checked
            }
        }));
    } else {
        // Gestion normale des autres champs
        setFormData(prev => ({
            ...prev,
            [name]: type === 'checkbox' ? checked : value
        }));
    }
    
    if (errors[name]) {
        setErrors(prev => ({ ...prev, [name]: '' }));
    }
};
