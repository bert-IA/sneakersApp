TUTORIEL REACT : FILTRAGE DES SNEAKERS PAR MARQUE
================================================

PROBLÉMATIQUE : FILTRAGE DYNAMIQUE DES DONNÉES
===============================================

OBJECTIF
--------
Créer un système de filtrage qui permet à l'utilisateur de sélectionner une marque dans une liste déroulante et d'afficher uniquement les sneakers de cette marque en temps réel.

CONCEPTS REACT IMPLIQUÉS
------------------------
✓ Lifting State Up (remontée d'état)
✓ Calculs dérivés et données filtrées
✓ Communication parent-enfant via props et callbacks
✓ Extraction de données avec reduce() ou Set
✓ Rendu conditionnel basé sur l'état
✓ Optimisation avec useMemo (optionnel)

ARCHITECTURE PROPOSÉE
=====================

STRUCTURE DES COMPOSANTS
------------------------
App (gère selectedBrand + logique de filtrage)
├── BrandFilter (liste déroulante des marques)
├── ShoppingList (affiche sneakers filtrés)
│   └── SneakerItem
└── Cart (inchangé)

FLUX DE DONNÉES COMPLET
=======================

ÉTAPE 1 : SÉLECTION DE MARQUE
-----------------------------
Utilisateur sélectionne une marque dans BrandFilter
↓
onChange déclenche handleChange dans BrandFilter
↓
BrandFilter appelle onBrandChange(selectedValue)

ÉTAPE 2 : REMONTÉE VERS APP
---------------------------
App reçoit la nouvelle marque via onBrandChange
↓
App met à jour son état selectedBrand avec setState
↓
React détecte le changement d'état

ÉTAPE 3 : CALCUL DES DONNÉES FILTRÉES
-------------------------------------
App recalcule filteredSneakers basé sur selectedBrand
↓
Logique : si "all" → tous les sneakers, sinon filter par marque
↓
Nouvelles données calculées automatiquement

ÉTAPE 4 : RE-RENDU AUTOMATIQUE
------------------------------
React re-rend ShoppingList avec les nouvelles données
↓
Seuls les sneakers de la marque sélectionnée s'affichent
↓
Interface mise à jour instantanément

ÉTAPE 1 : MODIFICATION DE APP.JS
--------------------------------
Objectif : Ajouter la gestion du filtrage par marque

CODE À AJOUTER/MODIFIER :
```javascript
import { useState } from 'react';
import ShoppingList from './Components/ShoppingList';
import Cart from './Components/Cart';
import BrandFilter from './Components/BrandFilter';  // Nouveau composant
import sneakersList from './data/sneakersList';

function App() {
    // États existants
    const [cart, setCart] = useState([]);
    
    // Nouvel état pour le filtrage
    const [selectedBrand, setSelectedBrand] = useState('all');
    
    // Fonction existante
    const addToCart = (sneaker) => {
        // ... logique addToCart existante
    };
    
    // Nouvelle fonction pour changer le filtre
    const handleBrandChange = (brand) => {
        setSelectedBrand(brand);
    };
    
    // Calcul des sneakers filtrés
    const filteredSneakers = selectedBrand === 'all' 
        ? sneakersList 
        : sneakersList.filter(sneaker => sneaker.marque === selectedBrand);
    
    return (
        <div className="App">
            <BrandFilter 
                selectedBrand={selectedBrand}
                onBrandChange={handleBrandChange}
                sneakers={sneakersList}
            />
            <ShoppingList 
                sneakers={filteredSneakers}  // Données filtrées
                onAddToCart={addToCart} 
            />
            <Cart cartItems={cart} />
        </div>
    );
}

export default App;
```

CONCEPTS CLÉS EXPLIQUÉS :
- selectedBrand : État pour stocker la marque sélectionnée
- handleBrandChange : Fonction callback pour mettre à jour le filtre
- filteredSneakers : Calcul dérivé basé sur selectedBrand
- Props transmission : sneakers au lieu de sneakersList statique

ÉTAPE 2 : CRÉATION DU COMPOSANT BRANDFILTER.JS
----------------------------------------------
Objectif : Créer un composant de filtrage réutilisable

CODE À CRÉER (nouveau fichier) :
```javascript
import React from 'react';
import './BrandFilter.css';

function BrandFilter({ selectedBrand, onBrandChange, sneakers }) {
    // Méthode 1 : Extraction des marques uniques avec reduce
    const brands = sneakers.reduce((acc, sneaker) => {
        if (!acc.includes(sneaker.marque)) {
            acc.push(sneaker.marque);
        }
        return acc;
    }, []);
    
    // Méthode 2 : Alternative avec Set (plus élégant)
    // const brands = [...new Set(sneakers.map(sneaker => sneaker.marque))];
    
    // Gestionnaire de changement
    const handleChange = (e) => {
        onBrandChange(e.target.value);
    };
    
    return (
        <div className="brand-filter">
            <label htmlFor="brand-select">Filtrer par marque :</label>
            <select 
                id="brand-select"
                value={selectedBrand} 
                onChange={handleChange}
                className="brand-select"
            >
                <option value="all">Toutes les marques</option>
                {brands.map(brand => (
                    <option key={brand} value={brand}>
                        {brand}
                    </option>
                ))}
            </select>
        </div>
    );
}

export default BrandFilter;
```

CONCEPTS CLÉS :
- reduce() : Extraction de valeurs uniques d'un tableau
- Set : Alternative moderne pour valeurs uniques
- Controlled component : value et onChange liés
- map() : Génération dynamique des options

OPTIMISATIONS AVANCÉES (OPTIONNEL)
==================================

PERFORMANCE AVEC USEMEMO
------------------------
Pour éviter les recalculs inutiles sur de gros datasets :

```javascript
// Dans App.js
import { useState, useMemo } from 'react';

function App() {
    const [selectedBrand, setSelectedBrand] = useState('all');
    
    // Optimisation : mémoriser les sneakers filtrés
    const filteredSneakers = useMemo(() => {
        return selectedBrand === 'all' 
            ? sneakersList 
            : sneakersList.filter(sneaker => sneaker.marque === selectedBrand);
    }, [selectedBrand]);
    
    // Optimisation : mémoriser les marques uniques
    const uniqueBrands = useMemo(() => {
        return [...new Set(sneakersList.map(sneaker => sneaker.marque))];
    }, []); // Tableau vide = calcul une seule fois
    
    return (
        <div className="App">
            <BrandFilter 
                selectedBrand={selectedBrand}
                onBrandChange={handleBrandChange}
                brands={uniqueBrands}  // Passer directement les marques
            />
            {/* ... reste du code */}
        </div>
    );
}
```

MULTI-FILTRES (EXTENSION FUTURE)
================================

STRUCTURE D'ÉTAT ÉTENDUE
------------------------
Pour gérer plusieurs filtres simultanément :

```javascript
// État plus complexe
const [filters, setFilters] = useState({
    brand: 'all',
    priceRange: 'all',
    style: 'all',
    bestSeller: false
});

// Fonction de mise à jour
const updateFilter = (filterType, value) => {
    setFilters(prev => ({
        ...prev,
        [filterType]: value
    }));
};

// Calcul de filtrage complexe
const filteredSneakers = useMemo(() => {
    return sneakersList.filter(sneaker => {
        const matchesBrand = filters.brand === 'all' || 
                           sneaker.marque === filters.brand;
        
        const matchesStyle = filters.style === 'all' || 
                           sneaker.style === filters.style;
        
        const matchesBestSeller = !filters.bestSeller || 
                                sneaker.bestSeller;
        
        return matchesBrand && matchesStyle && matchesBestSeller;
    });
}, [filters]);
```
