SYNTHÈSE PÉDAGOGIQUE : CENTRALISATION ET CALCUL DYNAMIQUE DES PRIX DANS UNE APPLICATION REACT

1. CENTRALISATION DES PRIX

- Créez un fichier src/datas/orderOptions.js pour centraliser les prix des options de livraison et des services additionnels.
- Exemple de contenu :
  export const DELIVERY_OPTIONS = {
    standard: { label: "Standard (3-5 jours)", price: 0 },
    express: { label: "Express (24h)", price: 9.99 }
  };

  export const ADDITIONAL_SERVICES = {
    giftWrap: { label: "Emballage cadeau", price: 5 },
    insurance: { label: "Assurance livraison", price: 15 },
    assembly: { label: "Montage", price: 105 }
  };

- Avantage : toute modification de prix ou d’intitulé se fait dans un seul fichier, ce qui simplifie la maintenance.

2. AFFICHAGE DYNAMIQUE DES OPTIONS DANS LE FORMULAIRE

- Dans ShippingPage.jsx, importez les objets de prix :
  import { ADDITIONAL_SERVICES } from '../datas/orderOptions';

- Utilisez ces objets pour générer dynamiquement l’affichage des cases à cocher :
  {`${ADDITIONAL_SERVICES.assembly.label} (+${ADDITIONAL_SERVICES.assembly.price},00€)`}

- Cela garantit que l’affichage et le calcul seront toujours synchronisés avec les valeurs de orderOptions.js.

3. PASSAGE DES CHOIX SÉLECTIONNÉS À LA PAGE DE RÉCAPITULATIF

- Pour que CheckoutPage puisse afficher et calculer le total avec les options choisies, il faut lui passer en props :
  - selectedDelivery (clé de l’option livraison choisie, ex : "express")
  - selectedServices (objet de type { assembly: true, giftWrap: false, insurance: true })

- Exemple d’appel :
  <CheckoutPage cart={cart} cartItemsCount={cartItemsCount} selectedDelivery={selectedDelivery} selectedServices={selectedServices} />

4. CALCUL DYNAMIQUE DU TOTAL DANS CheckoutPage.jsx

- Importez les prix centralisés :
  import { DELIVERY_OPTIONS, ADDITIONAL_SERVICES } from '../datas/orderOptions';

- Calculez le sous-total du panier :
  const sousTotal = cart.reduce((total, item) => total + (item.prix * item.quantity), 0);

- Calculez le prix de la livraison :
  const deliveryPrice = DELIVERY_OPTIONS[selectedDelivery]?.price || 0;

- Calculez le prix des services additionnels :
  const servicesPrice = Object.entries(selectedServices)
    .filter(([key, isChecked]) => isChecked)
    .reduce((sum, [key]) => sum + (ADDITIONAL_SERVICES[key]?.price || 0), 0);

- Calculez le total général :
  const totalHT = (sousTotal + deliveryPrice + servicesPrice) / 1.2;
  const tva = (sousTotal + deliveryPrice + servicesPrice) - totalHT;
  const totalTTC = sousTotal + deliveryPrice + servicesPrice;

5. AFFICHAGE DU DÉTAIL DANS LE RÉCAPITULATIF

- Affichez la livraison choisie et les services additionnels sélectionnés avec leur prix :
  Livraison : {DELIVERY_OPTIONS[selectedDelivery]?.label} (+{deliveryPrice}€)
  Pour chaque service sélectionné :
    {ADDITIONAL_SERVICES[key]?.label} : +{ADDITIONAL_SERVICES[key]?.price}€

- Affichez le total mis à jour dans la section récapitulative.

6. BONNES PRATIQUES

- Utilisez toujours la clé (ex : "assembly") pour faire le lien entre le state, le formulaire et le calcul.
- Le label ne sert qu’à l’affichage.
- Le prix est utilisé pour le calcul.
- Toute modification de prix ou d’intitulé se fait dans orderOptions.js.

7. RÉSUMÉ DU FLUX

- L’utilisateur coche des options dans ShippingPage.jsx.
- Les choix sont stockés dans le state et transmis à CheckoutPage.jsx.
- CheckoutPage.jsx importe les prix centralisés, calcule dynamiquement le total, et affiche le détail des options choisies.
- Le code reste maintenable, évolutif et cohérent.
